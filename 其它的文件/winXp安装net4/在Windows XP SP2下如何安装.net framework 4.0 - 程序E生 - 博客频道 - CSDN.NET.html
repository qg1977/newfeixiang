在Windows XP SP2下如何安装.net framework 4.0

2013-05-09 17:57 2469人阅读 评论(0) 收藏 举报
本文章已收录于：
分类：

版权声明：本文为博主原创文章，未经博主允许不得转载。

在微软的官方网站上针对.net framework 4.0的系统要求是这样描述的（http://www.microsoft.com/zh-cn/download/details.aspx?id=17718）

  • 支持的操作系统

    Windows 7, Windows 7 Service Pack 1, Windows Server 2003 Service Pack 2, Windows Server 2008, Windows Server 2008 R2, Windows Server 2008 R2 SP1, Windows Vista Service Pack 1, Windows XP Service Pack 3

      • Windows XP SP3
      • Windows Server 2003 SP2
      • Windows Vista SP1 或更高版本
      • Windows Server 2008（在服务器核心角色上不受支持）
      • Windows 7
      • Windows Server 2008 R2（在服务器核心角色上不受支持）
      • Windows 7 SP1
      • Windows Server 2008 R2 SP1
      • 支持的体系结构：
        • x86
        • x64
        • ia64（有些功能在诸如 WPF 之类的 ia64 上不受支持）
      • 硬件要求：
        • 建议的最低要求：Pentium 1 GHz 或更快，512 MB RAM 或更大
        • 最小磁盘空间：
          • x86 – 850 MB
          • x64 – 2 GB
      • 必备组件：
        • Windows Installer 3.1 或更高版本
        • Internet Explorer 5.01 或更高版本
很明显Windows XP SP2 无法安装.net framework 4.0，但是开发的程序是基于.net framework 4.0，客户的电脑系统是Windows XP SP2的，让客户更换系统简直是难上加难，经过查询资料，这个问题迎刃而解，在Windows XP SP2的电脑上首先安装wic_x68_chs.exe，这个文件可以从网上下载到，之后再安装.net framework 4.0就没有任何问题了。
